# Project conventions for Claude

## Commit messages and pull requests

Do not include co-author trailers, session identifiers, or tool
attribution in commit messages or pull request bodies. Specifically:

- No `Co-Authored-By:` trailers (including `Co-Authored-By: Claude ...`).
- No `Claude-Session:` lines or any other session-URL trailer.
- No "Generated by / Generated with Claude Code" footers, in any form —
  plain text, markdown link (`_Generated by [Claude Code](...)_`), or
  with a leading `🤖` or `---` separator.
- No `https://claude.ai/code/...` URLs pasted into commits or PR bodies.

If the harness auto-appends any of the above after a `create_pull_request`
call, remove it immediately with an `update_pull_request` call so the PR
body ends where the human-authored content ends.

Write the commit or PR body as if a human maintainer authored it — the
subject/body should describe the change and the reasoning, nothing else.
